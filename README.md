# Правила пользования репозиторием tages-shared-files

Этот репозиторий предназначен для обмена файлами между участниками команды.

## Основные правила

1. **Создайте свою личную директорию в корне репозитория**  
   Название директории: `<Фамилия>_<Имя>` — **ТОЛЬКО ЛАТИНИЦЕЙ и БЕЗ ПРОБЕЛОВ**  
   Пример: `Khakimov_Nikita`

   Внутри этой директории размещайте все свои файлы и поддиректории. Структура внутри вашей папки - на ваше усмотрение, но старайтесь делать её понятной для других.

2. **Запрещено редактировать или удалять содержимое чужих директорий**  
   Если вам нужно изменить чужой файл — скопируйте его к себе в директорию и работайте с копией.

3. **Не добавляйте в репозиторий лишние файлы от ide и тд.**  
   Используйте `.gitignore` (уже есть в репозитории) и проверяйте `git status` перед коммитом.

## Как начать работу

```bash
# 1. Клонируем репозиторий (один раз)
git clone https://github.com/NikosEvenSecond/tages-shared-files.git
cd tages-shared-files

# 2. Создаём свою директорию (обязательно латиницей!)
mkdir <Фамилия>_<Имя>
# Пример:
mkdir Khakimov_Nikita

# 3. Кладём туда свои файлы, создаём нужную структуру
```

## Как правильно запушить изменения
```bash 
# 1. Добавляем ТОЛЬКО свою директорию (или файлы внутри неё)
git add <Название вашей директории>
# Пример:
git add Khakimov_Nikita

# 2. Проверяем, что не попало ничего лишнего
git status

# 3. Делаем понятный коммит
git commit -m "<Название вашей директории>: <что сделали>"
# Пример:
git commit -m "Khakimov_Nikita: add ER-diagram"

# 4. Подтягиваем чужие изменения
git pull --rebase

# 5. Пушим свои изменения
git push
```
